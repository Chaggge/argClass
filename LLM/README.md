# README: LLM для классификации аргументации (RuadaptQwen3-4B)

## Описание проекта

Данный Jupyter Notebook реализует двухэтапную классификацию аргументации с использованием малых генеративных моделей (LLM) в режимах zero-shot и few-shot. Проект включает:

- Бинарную классификацию (атакующие vs. поддерживающие аргументы)
- Многоклассовую классификацию типов атакующих и конфликтных схем
- Стратифицированную кросс-валидацию с группировкой по текстам
- Отбор примеров для few-shot промптов (случайный / семантически близкий)
- Кэширование эмбеддингов для ускорения работы

## Структура ноутбука

### 1. Импорт библиотек и настройка параметров
Загрузка необходимых библиотек (`transformers`, `sentence-transformers`, `sklearn`, `pandas`, `tqdm`). Определение глобальных параметров: размер батча, seed, пути к данным.

### 2. Загрузка и подготовка данных
- Чтение CSV-файлов (`inf_8.csv`, `conf_8.csv`)
- Функция `newClasses()` — объединение inference и conflict данных, создание бинарной метки `attack`
- Подготовка DataFrame для LLM через `prepare_df_for_llm()`:
  - Форматирование текста: `premise: "..." \n conclusion: "..."`
  - Фильтрация редких схем (по умолчанию `min_examples_per_scheme=10`)

### 3. Инициализация LLM
Используется пайплайн `text-generation` из библиотеки `transformers`. Основная модель: `RefalMachine/RuadaptQwen3-4B-Instruct` (адаптирована для русского языка). Поддерживаются также:

### 4. Вспомогательные функции
- `llm_generate_fn()` — массовая генерация ответов батчами
- `clean_output()` — очистка и валидация ответов модели
- `remove_tags()`, `ensureDot()` — текстовая предобработка

### 5. Ретривер для few-shot примеров
- Модель эмбеддингов: `paraphrase-multilingual-MiniLM-L12-v2` (поддержка русского языка)
- `EmbeddingCache` — кэширование эмбеддингов на диск (файл `retriever_cache.pkl`)
- `build_retrieval_index()` — построение индекса обучающих примеров
- `retrieve_few_shot()` — поиск топ-k семантически близких примеров для запроса
- `retrieve_few_shot_random()` — случайный отбор примеров (для сравнения)

### 6. Промпты для LLM
#### Бинарная классификация (`build_attack_prompt`)
- Задача: определить, является ли аргумент атакующим (ATTACK) или поддерживающим (SUPPORT)
- Few-shot примеры вставляются в промпт
- Ответ извлекается через `map_attack_label()`

#### Многоклассовая классификация (`build_scheme_prompt`)
- Задача: классифицировать атакующий аргумент по типам схем (11 классов)
- Встроены описания классов (`SCHEME_DESCRIPTIONS`)
- Правила: сначала определить атака/поддержка, затем выбрать схему
- Принимает few-shot примеры (текст → схема)

### 7. Кросс-валидация
`crossval_split_llm()` — стратифицированное разбиение по текстам (мультилейбл стратификация). Сохраняется группировка по `content`, примеры из одного текста не попадают в разные фолды.

### 8. Двухэтапная классификация
`run_crossval_llm_2stage()` — основной цикл:

**Stage 1 (Attack/Support):**
- Построение индекса для `attack`
- Для каждого текста в валидации — поиск few-shot примеров
- Генерация ответа через LLM, извлечение метки
- Сохранение предсказаний

**Stage 2 (Scheme):**
- Фильтрация примеров, где `preds_atk == '1'` и `true_atk == '1'`
- Построение индекса для `scheme` на обучающих атакующих примерах
- Для каждого такого примера — few-shot промпт с примерами схем
- Генерация и извлечение схемы

### 9. Выполнение
```python
df = prepare_df_for_llm(u_inf_df)
run_crossval_llm_2stage(df, n_splits=3, pck1=2, pck2=2)
```
