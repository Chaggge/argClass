# README: Парсинг и формирование датасета ArgNetSC

## Описание проекта

Данный Jupyter Notebook предназначен для загрузки, парсинга и преобразования сырых JSON-файлов аннотированного корпуса ArgNetSC в структурированные CSV-файлы для дальнейшего использования в задачах классификации аргументации.

Проект выполняет:
- Загрузку всех JSON-файлов из папки `data`
- Рекурсивный парсинг аргументативных структур (Inference и Conflict)
- Формирование двух отдельных DataFrame: `inference_df` и `conflict_df`
- Сохранение результатов в CSV-файлы (`inf_8.csv`, `conf_8.csv`)

---

## Структура ноутбука

### 1. Импорт библиотек
- `pandas` — работа с DataFrame
- `json` — чтение JSON-файлов
- `os` — работа с файловой системой
- `re` — регулярные выражения
- `tqdm` — отображение прогресса

### 2. Загрузка данных
Функция `loadJsonData(path_to_data)` загружает все JSON-файлы из указанной папки и возвращает список словарей.

### 3. Вспомогательные функции

| Функция | Назначение |
|---------|------------|
| `make_base_dict()` | Создание словаря-шаблона для хранения данных (uri, content, range, type, comments) |
| `project_parser()` | Извлечение statements и arguments из проекта, заполнение базового словаря |
| `statement_check()` | Рекурсивное извлечение текста утверждения по URI. Если аргумент вложенный, раскрывается его структура (посылки и заключение) |

### 4. Основная функция парсинга
`parseData(data)` — рекурсивно обходит структуру корпуса:

- Извлекает тексты (`content`) и комментарии
- Для каждого `project`:
  - Создаёт `base_df` со всеми statements и arguments
  - Разделяет аргументы на два типа:
    - **Conflict** — конфликтные схемы (с посылками и исключениями)
    - **Inference** — умозаключения (с посылками и заключением)
- Возвращает два DataFrame: `inference_df`, `conflict_df`

### 5. Запуск парсинга

```python
# Загрузка всех JSON-файлов
data_collection = loadJsonData("./data")

# Парсинг каждого файла
parsed_data = [parseData(data) for data in data_collection]

# Обработка пропусков и сортировка колонок

### Особенности обработки данных

#### 1. Заполнение пропусков
Все пропущенные значения (NaN) заполняются строкой `'NaN'`. Это обеспечивает единообразие типов данных в DataFrame и предотвращает ошибки при последующей обработке.

#### 2. Сортировка колонок
Колонки в выходных DataFrame автоматически сортируются по алфавиту для обеспечения консистентности структуры при каждом запуске.

#### 3. Обработка вложенных аргументов
- Если аргумент ссылается на другой аргумент (`is_statement = False`), функция `statement_check()` рекурсивно раскрывает его структуру
- Результат собирается из посылок и заключения вложенного аргумента
- Это позволяет сохранить полную цепочку рассуждений без потери информации

#### 4. Тег `<del>`
При объединении нескольких фрагментов текста (например, нескольких посылок или заключений) они разделяются тегом `<del>`. Пример:

#### 5. Форматирование комментариев
Все комментарии к тексту сохраняются с префиксом `comment N:`, где `N` — порядковый номер комментария. Несколько комментариев разделяются двумя символами новой строки (`\n\n`):


#### 6. Разделение по типу аргументации
Аргументы разделяются на два типа по последней части названия схемы (после символа `_`):
- `..._Inference` → попадает в `inference_df`
- `..._Conflict` → попадает в `conflict_df`

Название схемы сохраняется полностью в колонке `scheme`.

#### 7. Динамическое количество колонок
Количество колонок `Premise_N`, `Exception_N` зависит от максимального числа посылок или исключений в конкретном аргументе. Например:
- Аргумент с 2 посылками получит колонки `Premise_0`, `Premise_1`
- Аргумент с 1 исключением получит колонку `Exception_0`

#### 8. Рекурсивный обход структуры
Функция `parseData()` рекурсивно обходит вложенные структуры `subcorpora`, что позволяет корректно обрабатывать данные любой глубины вложенности.

#### 9. Прогресс-бар
Для отслеживания процесса парсинга используется библиотека `tqdm`, отображающая прогресс-бар при обработке каждого файла.

#### 10. Целостность данных
Перед сохранением данные проходят проверку:
- Удаляются дублирующиеся `uri`
- Проверяется наличие обязательных полей
- Выводится статистика о количестве обработанных аргументов
