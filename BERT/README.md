# README: Классификация аргументации в полемических текстах (BERT)

## Описание проекта

Данный Jupyter Notebook реализует двухэтапную классификацию аргументации в русскоязычных полемических текстах на основе таксономии Д. Уолтона. Проект включает:

- Бинарную классификацию (атакующие vs. поддерживающие аргументы)
- Многоклассовую классификацию типов атакующих и конфликтных схем
- Стратифицированную кросс-валидацию с группировкой по текстам
- Методы предобработки данных и борьбы с дисбалансом классов

---

## Структура ноутбука

### 1. Импорт библиотек и настройка параметров
Загрузка необходимых библиотек (`datasets`, `transformers`, `sklearn`, `pandas`, `torch`). Определение глобальных параметров: размер батча, seed, максимальное количество токенов, пути к данным.

### 2. Загрузка модели и токенизатора
Используемая модель: `DeepPavlov/rubert-base-cased` (RuBERT). Добавлены специальные токены: `<Premise>`, `</Premise>`, `<Conclusion>`, `</Conclusion>`.

### 3. Формирование датасета
Функция `newClasses()` объединяет inference и conflict данные, создаёт бинарную метку `attack` (1 — атакующий, 0 — поддерживающий). Все конфликтные схемы объединены в класс `Conflict`.

### 4. Предобработка текста (дополнение контекстом)
Функция `contex2df()` находит посылки и заключения в исходных текстах (комментариях или основном контенте), выделяет целые абзацы, содержащие аргумент, и добавляет теги `<Premise>` и `<Conclusion>` для маркировки границ.

### 5. Подготовка данных для модели
- `prepare_Dataset()` — конкатенация всех текстовых полей в одну строку
- `preprocessFunction()` — объединение с разделителем `[SEP]`
- `tokenizeAndLabeling()` — токенизация и добавление меток классов

### 6. Метрики и callback
- `compute_metrics()` — расчёт accuracy, precision, recall, F₁ (macro avg)
- `LossLoggerCallback()` — логирование и визуализация функции потерь

### 7. Методы борьбы с дисбалансом классов
- `oversample_df()` — дублирование примеров редких классов
- `compute_class_weights_from_dataset()` — расчёт весов классов по формуле `balanced`
- `FocalLoss` — реализация фокальной функции потерь (γ = 2)
- `WeightedTrainer` — кастомный тренер с поддержкой class weights и Focal Loss

### 8. Кросс-валидация
`cross_validation_load_prepare_data()` выполняет стратифицированное разбиение по текстам (5 фолдов). Гарантируется, что примеры из одного текста не попадают в разные выборки. Проверяется присутствие всех классов в train и val.

### 9. Обучение и оценка
- `run_model_fold_classification_report()` — обучение и оценка для одного фолда
- `run_crossval_2stage()` — двухэтапная кросс-валидация:
  - **Stage 1**: бинарная классификация (Attack vs. Support)
  - **Stage 2**: многоклассовая классификация типов атак (только для примеров, предсказанных как атакующие)

## Используемые методы

| Метод | Назначение |
|-------|------------|
| `contex2df()` | Дополнение аргументов контекстом (целый абзац) |
| Теги `<Premise>`/`<Conclusion>` | Явное выделение границ посылок и заключений |
| `oversample_df()` | Борьба с дисбалансом (дублирование редких классов) |
| `WeightedTrainer` | Взвешивание классов при расчёте loss |
| `FocalLoss` | Фокальная функция потерь (γ = 2) |
| Стратифицированная кросс-валидация | Оценка на 5 фолдах с группировкой по текстам |
